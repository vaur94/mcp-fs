# Security Policy

## Supported versions
- `0.1.x`: supported

## Reporting a vulnerability
Please report vulnerabilities privately.

- Open a private security advisory in GitHub, or
- Email the maintainers listed in repository settings

Include:
- Affected version
- Reproduction steps
- Impact assessment
- Suggested mitigation (if known)

## Response targets
- Initial response: within 5 business days
- Triage decision: within 10 business days
- Fix timeline: depends on severity and exploitability

## Scope notes
- `mcp-fs` is local-first and offline by design.
- Security issues around path traversal, root escape, symlink handling, and patch integrity are high priority.
